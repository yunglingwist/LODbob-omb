RDF Production

The primary objective and final technical milestone of this project was the transformation of heterogeneous data sources into a unified RDF format. Every object in our collection—whether stored in CSV or other digital formats—was systematically converted into a machine-readable Linked Data format.


To achieve this, we developed a robust Python-based pipeline (rdf_convert.py) tailored to the specific structure of each input file. The transformation process followed a rigorous methodology:


  • URI Minting: Identifying and assigning unique subject URIs within the https://scottpilgrim.org/resource/ namespace.
  • Predicate Mapping: Aligning data fields with established ontologies such as Schema.org, FOAF, Dublin Core, Music Ontology, and Relationship Ontology.
  • Data Typing: Carefully differentiating between Literals (names, years, descriptions) and Resources (linking to other internal URIs or external entities), ensuring compatibility with the domain and range of the chosen properties.
  • Semantic Enrichment: Building contextual layers by connecting characters to their musical instruments, social relationships, and geographical locations.

For the majority of our datasets—including the Anime (ANIM), Characters (PPL), Instruments (GTR), and Soundtracks (SNDTRCK)—we utilized CSV format as the intermediary storage. The logic behind our Python transformation was modular: for each row, the script parsed the predicate, resolved it through a prefix mapping, and determined the object type. We ensured that all relevant namespaces were bound to the RDF graph for standardization and readability.


All resulting graphs were serialized in Turtle (.ttl) format. This format was chosen for its high human-readability and its clear representation of the Subject–Predicate–Object pattern, making the data easily queryable via SPARQL. The technical stack relied on RDFLib for graph construction and serialization, maintaining a shared structure across all scripts to ensure overall coherence.

Data Orchestration: From Heterogeneous Sources to RDF

Below is the example of our Python pipeline used for RDF conversion and namespace management.


# ======== Base Configuration ========= #
import csv
import os
from rdflib import Graph, Namespace, URIRef, Literal, RDF, XSD

# All Namespaces
NS = {
    "sp": Namespace("https://scottpilgrim.org/resource/"),
    "bf": Namespace("http://id.loc.gov/ontologies/bibframe/"),
    "dcterms": Namespace("http://purl.org/dc/terms/"),
    "foaf": Namespace("http://xmlns.com/foaf/0.1/"),
    "gn": Namespace("http://www.geonames.org/ontology#"),
    "owl": Namespace("http://www.w3.org/2002/07/owl#"),
    "rdfs": Namespace("http://www.w3.org/2000/01/rdf-schema#"),
    "schema": Namespace("https://schema.org/"),
    "skos": Namespace("http://www.w3.org/2004/02/skos/core#"),
    "mo": Namespace("http://purl.org/ontology/mo/"),
    "rel": Namespace("http://purl.org/vocab/relationship/"),
    "rdf": Namespace("http://www.w3.org/1999/02/22-rdf-syntax-ns#"),
    "org": Namespace("http://www.w3.org/ns/org#")
}

def init_graph():
    g = Graph()
    for prefix, ns in NS.items():
        g.bind(prefix, ns)
    return g

g = init_graph()

def get_sp_uri(val):
    if not val: return None
    return NS["sp"][val.replace("local:", "").strip()]

# ======== INDIVIDUAL FILE PROCESSING (MANUAL) ========= #

# 1. PPL.csv (Characters & Persons)
if os.path.exists("PPL.csv"):
    with open("PPL.csv", newline='', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        for row in reader:
            # Subject definition
            sub = NS["sp"][row["id"]]
            
            # 1. rdf:type 
            if row.get("rdf:type"):
                type_prefix, type_term = row["rdf:type"].split(':')
                g.add((sub, RDF.type, NS[type_prefix][type_term]))
            else:
                # Fallback logic if rdf:type is empty
                g.add((sub, RDF.type, NS["schema"]["FictionalCharacter"] if "char_" in row["id"] else NS["foaf"]["Person"]))

            # 2. foaf:name (Literal)
            if row.get("foaf:name"):
                g.add((sub, NS["foaf"]["name"], Literal(row["foaf:name"])))

            # 3. schema:roleName (Literal - Protagonist, Support etc.)
            if row.get("schema:roleName"):
                g.add((sub, NS["schema"]["roleName"], Literal(row["schema:roleName"])))

            # 4. schema:portrayedBy (URI - Linked to another Person)
            if row.get("schema:portrayedBy"):
                g.add((sub, NS["schema"]["portrayedBy"], get_sp_uri(row["schema:portrayedBy"])))

            # 5. org:memberOf (URI - Linked to a Group/Band)
            if row.get("org:memberOf"):
           
                for group in row["org:memberOf"].split(';'):
                    g.add((sub, NS["org"]["memberOf"], get_sp_uri(group)))

            # 6. owl:sameAs (URIRef - Wikidata/External Links)
            if row.get("owl:sameAs"):
                g.add((sub, NS["owl"]["sameAs"], URIRef(row["owl:sameAs"])))

            # 7. foaf:page (URIRef - Fandom/Web pages)
            if row.get("foaf:page"):
                g.add((sub, NS["foaf"]["page"], URIRef(row["foaf:page"])))

            # 8. dcterms:isPartOf (URI - Linked to the Universe)
            if row.get("dcterms:isPartOf"):
                g.add((sub, NS["dcterms"]["isPartOf"], get_sp_uri(row["dcterms:isPartOf"])))

RDF Visualization

An RDF (Resource Description Framework) representation is a structured method for describing data through triples: subject, predicate, and object. These triples form a directed graph where nodes represent resources or literal values, and edges represent the semantic relationships between them. This framework enables our Scott Pilgrim dataset to be linked and understood across disparate systems, serving as a core component of the Semantic Web.


Following our theoretical and conceptual models, we described each of our items in RDF Turtle (.ttl) format. We assigned a unique URI to every object and entity to ensure seamless connectivity within our ontology. To validate the structural integrity and visual coherence of our work, we utilized RDF Grapher. This tool allowed us to transform our Turtle code into visual representations, confirming that the relationships between characters, media formats, and real-world locations were correctly mapped as an interconnected network.


Figure 1: Visual representation of the Scott Pilgrim Knowledge Graph generated via RDF Grapher.

Finally, we compiled all individual Turtle files into a single, comprehensive dataset. This unified representation acts as our final model, where fictional narratives, musical inspirations, and geographic locations coexist in a navigable and machine-interpretable knowledge platform.
